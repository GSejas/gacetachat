# ASCII Mockup Standards for UI/UX Documentation

## Overview

This document establishes standardized conventions for creating ASCII art mockups in UI/UX documentation, ensuring consistency, readability, and effective communication of design concepts.

## Core Principles

1. **Consistency**: Use standardized symbols and layouts across all mockups
2. **Clarity**: Ensure mockups are easily readable and interpretable
3. **Comparison**: Always show before/after when demonstrating improvements
4. **Annotation**: Include clear explanations and callouts for design decisions
5. **Accessibility**: Consider how mockups will be read by screen readers

### Do's âœ…

- Include clear before/after comparisons
- Add descriptive callouts with arrows
- Show user emotional states
- Include mobile variations
- Use standard symbols for common UI elements
- Maintain consistent spacing and alignment

### Don'ts âŒ

- Create mockups without explanations
- Use unclear or inconsistent symbols
- Skip mobile considerations

- Use overly complex layouts
- Ignore accessibility considerations

### Accessibility Considerations
- Use descriptive text alongside visual elements
- Ensure mockups are readable by screen readers
- Include alternative text descriptions for complex diagrams
- Use clear heading hierarchy
- Provide context for all visual elements

- Online ASCII art generators
- Text editors with monospace fonts
- Unicode character picker tools
- Screen reader testing tools

### Character Reference
- Box Drawing: U+2500-U+257F
- Arrows: U+2190-U+21FF
- Symbols: U+2600-U+26FF
- Emoji: U+1F600-U+1F64F

This standard ensures all ASCII mockups in UI/UX documentation follow consistent, readable, and accessible formatting while effectively communicating design concepts and improvements.
